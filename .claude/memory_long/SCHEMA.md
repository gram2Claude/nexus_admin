# SCHEMA.md — мозг wiki-библиотекаря (проектная долгосрочная память Q2)

Схема проектной долгосрочной памяти (модель **«Wiki-LLM вместо RAG»**). Слэш-команды
`/ingest`, `/query`, `/lint` читают этот файл через `Read` **перед** работой — правила вики
не засоряют основной контекст разработки. Референс-спека: `11_short_memory/work_directory/01_specs/05_q2_long_term_spec.md`.

## 1. Роли
- **Человек-куратор:** кладёт сырьё в `raw/`, подтверждает планы, судит об истинности фактов.
- **LLM-библиотекарь:** читает сырьё, пишет и связывает страницы `wiki/`, ведёт индексы и журналы,
  отвечает на запросы, проверяет здоровье вики. **Не выдумывает факты; не пишет в базу без подтверждения.**

## 2. Структура (`.claude/memory_long/`)
- `SCHEMA.md` — этот файл (правила).
- `log.md` — append-only журнал **публичных** операций (ingest/query/lint).
- `raw/` — иммутабельные внешние источники (материалы), только чтение; `raw/_large/` — крупные/бинарные (gitignore).
- `wiki/` — генерируемые страницы со связями `[[...]]` (**плоско**, тип — во frontmatter):
  - `index.md` — каталог **публичных** страниц (коммитится).
  - `Карта проекта.md` — MOC верхнего уровня (хаб).
  - `План-факт.md` — `type=plan_fact` (витрина статуса; заводится при появлении плана).
  - `<материалы>.md` — `type=source`.
  - `_private/` — **приватное** (gitignore): дистиллят из диалогов (`type=concept|connection|qa`),
    со **своими** `_private/index.md` и `_private/log.md`.

## 3. Граница с продуктом (железно)
Файлы, читаемые кодом/сборкой/продом (конфиги, промпты, генерёжка, прод-доки), wiki-операции
**НЕ изменяют** и не добавляют в них frontmatter/теги. Вся работа — внутри `memory_long/`. Чтобы
отразить такой файл в графе — заведи **страницу-указатель** (`type=source`), которая описывает и
`[[ссылается]]`, но сам файл не правит.

## 4. Приватность (анти-утечка — критично)
- **Дистиллят из диалогов → только в `_private/`** (gitignore): растёт из приватных daily-логов.
- `_private/` ведёт **СВОИ** `index.md`/`log.md`. В **коммитимые** `wiki/index.md`, `wiki/log.md`,
  `Карта проекта.md` приватные страницы **НЕ попадают**.
- Связи **только приватное → публичное** (одностороннее). Обратные ссылки из публичного на приватное
  НЕ ставим — иначе заголовки/пути дистиллята утекут в git.
- PII/секреты не хранить в открытом виде — маскировать.

## 5. Формат страницы (обязателен — питает граф рёбрами)
```markdown
---
title: <Заголовок>
type: plan_fact | concept | connection | qa | source | moc
status: active | draft | deprecated
source: <путь в raw/ или daily/ | «—»>
tags: [...]
aliases: [...]
created: ГГГГ-ММ-ДД
updated: ГГГГ-ММ-ДД
---

# Заголовок
Тело (русский). Утверждения — с привязкой к источнику.

## Связи
- [[concepts/соседняя-страница]]
- [[Карта проекта]]   ← обязательная связь на хаб
```
Минимум 1 `[[ссылка]]`, иначе страница — **сирота**. Связи двунаправленные по смыслу (кроме границы §4).

## 6. Правила записи
- **Ссылки — полным путём от `wiki/`** (`[[concepts/slug]]`, не голый `slug`) — снимает коллизии имён.
- **Все файлы — UTF-8 без BOM.** Имена страниц допускают кириллицу/пробелы (`Карта проекта.md`).
- **«Черновик вместо провала»** — неполное помечать `status: draft` + `#черновик` + TODO, не выдумывать.
- `source` обязателен.
- Вики коммитить отдельными коммитами от кода.

## 7. INGEST (материалы; курируемо, write-gate)
**Правило по умолчанию: файл из `raw/` НЕ разбивается.** Один файл `raw/` → **одна цельная** страница
`type=source`: документ приводится к шаблону (frontmatter сверху + секция `## Связи` снизу), проставляются
связи, но **тело документа сохраняется целиком** (включая схемы Mermaid/таблицы — ничего не выкидываем и
не пересказываем). Разбивать материал на несколько страниц — **только** явной отдельной командой
`/ingest-split <файл>` (см. §11).

Шаги: 1. Прочитать `log.md` (что разобрано). 2. Найти новые файлы в `raw/` (`.md/.txt`/код → `Read`;
`.pdf` → `Read` по страницам ≤~20; `.docx` → `markitdown`, оригинал не трогать). 3. **ПОКАЗАТЬ ПЛАН и ждать
«да».** 4. На «да»: для каждого файла создать **одну** цельную `type=source` страницу (шаблон + связи на
хаб/соседей, тело целиком), обновить `index.md` + `log.md`. Граница §3 соблюдается.

## 8. QUERY (только чтение)
`index.md` **и** `_private/index.md` → рекурсивно по `[[ссылкам]]` → синтез **только по вики**, с
цитатами `[[страница]]`. Цитаты из `_private/` помечать. Нет ответа — сказать честно, предложить
`ingest`. Ничего не писать.

## 9. LINT (отчёт → правки по подтверждению)
Проверять: битые `[[ссылки]]`, сироты, недостающие связи (unlinked mentions), устаревшее, дубли,
рассинхрон индексов, нарушения формата. Отсутствие `_private/` (у склонировавшего без приватного) —
**не ошибка**. `source` на gitignored `raw/_large/` — **не ошибка**.

## 10. Точки входа
`index.md` (старт QUERY-public), `_private/index.md` (старт QUERY-private), `Карта проекта.md` (MOC),
`log.md` (старт INGEST), `SCHEMA.md` (правила).

## 11. Связь с движком
Дистиллят (`type=concept|connection|qa`) производит **`compile`** (движок, авто, пишет в `_private/`).
`query`/`lint` существуют и в движке (`scripts/query.py`, `scripts/lint.py`, портируются под per-project).
Слэш-команды (человеко-входные точки, читают этот SCHEMA): `/ingest` (1 файл → 1 цельная `type=source`
страница, без разбиения), `/ingest-split <файл>` (явное разбиение документа на атомарные страницы),
`/query` (чтение), `/lint` (гигиена).
