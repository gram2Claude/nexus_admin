# Конвенция работы с репозиторием nexus_admin

## Ветки

- **`master`** — защищённая целевая ветка. Прямой push запрещён (branch protection).
  Мержит только координатор (ИИ) через merge-гейт после зелёной проверки.
- **`oleg`** — ветка разработчика. Вся работа и все push — только сюда.

## Процесс разработчика

1. Берёшь задачу → пишешь в чат координатору: **«беру `NEXADM-N` в работу»**
   (N — номер задачи в Plane). Координатор ставит In Progress.
2. Работаешь в ветке `oleg`, коммитишь и пушишь как удобно — per-commit конвенция
   НЕ требуется (статусы выводятся не из коммитов). Желательно (для гигиены)
   упоминать `NEXADM-N` в сообщениях коммитов.
3. Готово → пишешь: **«готово к merge, `NEXADM-N`»** (можно несколько ID —
   спринт обычно поставляется одним merge).
4. Координатор: пробный merge → проверочная лестница (install, build, lint, смоук) →
   согласование с управленцем → merge `--no-ff` в `master` → задачи в Done.

## Проверка перед «готово к merge»

- `npm install && npm run build` — зелёные;
- линт без ошибок;
- критерии готовности задачи (done_criteria в Plane/каноне) выполнены.

## Артефакты планирования

`work_directory/00_global_plan/` — глобальный план (канон, витрина, ревью) — ведёт координатор,
руками не править. Спеки страниц — `work_directory/01_specs/`, планы реализации — `02_plans/`.
