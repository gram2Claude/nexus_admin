## 1. ASCII-схема страницы «Проекты» (t20) — НА СОГЛАСОВАНИЕ

Заменяет черновой список целиком. Один экран, две зоны: карточки + Гант.

```
┌─ Проекты ──────────────────────────────────────────────────────────────────────┐
│  [ Активные | Завершённые | Все ]                          ⟳ обновлено 5 мин   │
```

Условные обозначения и правила:
- **Переключатель** «Активные | Завершённые | Все» влияет на карточки И на Гант
  (завершённые на Ганте — приглушённым цветом).
- **Карточка**: имя, статус-точка, описание (2 строки, дальше многоточие), полоса прогресса
  фиксированной длины + % справа (как согласовано), часы план-факт; вторая строка факта:
  фактические часы · токены · ≈$ (видна только ролям с правом на затраты — Owner/Admin).
  ✏ — редактирование описания (Owner/Admin). Клик по карточке → drill-down проекта (эпоха 5).
- **Гант**: горизонталь — календарь (диапазон от min даты старта до max даты финиша
  активных проектов, деления по неделям); вертикаль — проекты.
  Полоса = плановый период проекта; **заливка ▓ = выполнено (% по часам), ░ = осталось**;
  % подписан слева у полосы. Вертикальные насечки внутри полосы — границы эпох.
- **Hover на сегмент эпохи** → тултип: название эпохи, плановые даты и часы,
  фактические часы, токены, ≈$ API-эквивалент (затраты — только Owner/Admin).
- **Линия «сегодня»** — вертикаль через весь Гант.
- Employee видит только свои проекты (membership) и без строк затрат; Client — заглушка.
- Данные: план — nexus_admin.projects/epochs (синк канонов), факт — v_task_fact roll-up
  по эпохам (готов в fact.ts). Всё уже в БД — мокать нечего.
