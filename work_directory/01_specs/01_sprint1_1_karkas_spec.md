## 1. t01 — Инициализация приложения: 3 варианта

**A. Минимальный:** `create-next-app` дефолтный + `shadcn init`. Быстро, но структура без `src/`,
без standalone-сборки (понадобится для Docker в эпохе 6) — доделывать потом.

**B. «Идеальный» (РЕКОМЕНДАЦИЯ):** `create-next-app` (TS, Tailwind 4, App Router, `src/`,
import-alias `@/*`) + `shadcn init` + сразу `output: "standalone"` в next.config (под Docker, эпоха 6)
+ ESLint + структура каталогов (`src/app`, `src/components`, `src/lib`, `src/server`).
Обоснование: +20 минут к варианту A, но снимает две переделки (Docker-сборка, структура), которые
точно понадобятся по плану.

**C. Латеральный:** стартовать с готового shadcn-блока `dashboard-01` (официальный admin-шаблон)
как базы layout. Ускоряет t04, НО тянет лишние компоненты и чужую структуру страниц — вычищать
дольше, чем собрать своё из тех же примитивов (sidebar-компонент shadcn возьмём и так).

> Так как t01 — обратимая инфраструктура и блокирует весь спринт, реализация начата по варианту B
> сразу; если выберешь другой вариант — переделаю до merge (стоимость правки минимальна).
ОТВЕТ: B. «Идеальный»

## 2. t03 — ASCII-схема общего layout кабинета (НА СОГЛАСОВАНИЕ)

Десктоп (основной сценарий — управленец за рабочим столом):

```
┌────────────┬──────────────────────────────────────────────────────────────────┐
│            │  ХЛЕБНЫЕ КРОШКИ: Проекты / nexus                 ⟳ обновлено 12м │
│  NEXUS     │                                              [🔔]  [Олег ▾ Owner]│
│  ADMIN     ├──────────────────────────────────────────────────────────────────┤
│            │                                                                  │
│ ────────── │                                                                  │
│ ▸ Проекты  │                                                                  │
│   Сотрудни-│                                                                  │
│   ки    ⏳ │                КОНТЕНТНАЯ ОБЛАСТЬ РАЗДЕЛА                        │
│   База зна-│                                                                  │
│   ний   ⏳ │     (страницы раздела «Проекты»: обзор, Гант, drill-down;       │
│   Отдел    │      разделы с ⏳ — заглушка «в разработке»)                     │
│   продаж ⏳│                                                                  │
│ ────────── │                                                                  │
│            │                                                                  │
│ Пользова-  │                                                                  │
│ тели    ⚙  │                                                                  │
│            │                                                                  │
│ [⟨] свернуть                                                                  │
└────────────┴──────────────────────────────────────────────────────────────────┘
```

Пояснения:
- **Sidebar слева** (сворачиваемый до иконок): логотип, 4 раздела; «Пользователи» — отдельным
  пунктом внизу (виден только Owner/Admin — это администрирование, не рабочий раздел).
- **Header**: хлебные крошки; **метка свежести данных** «⟳ обновлено N минут назад»
  (+ warning-баннер, если > 12 ч — решение ревью); меню пользователя (имя, роль, выход).
- **Контент** — остальное пространство; внутренняя навигация разделов (переключатель
  активные/завершённые и т.п.) живёт в контенте, не в sidebar.
- Планшет: sidebar автоматически сворачивается до иконок; телефон — вне скоупа первой очереди.

## 3. t02 — Дизайн-токены по референсу Logip (НА СОГЛАСОВАНИЕ)

По референсу Logip (светлая аналитика, мягкие карточки, один зелёный акцент):

| Токен | Значение | Где |
|---|---|---|
| Фон приложения | `#F6F7F9` (светло-серый) | подложка |
| Поверхности | `#FFFFFF`, радиус 12px, мягкая тень | карточки, sidebar, header |
| Текст основной | `#0F172A` (тёмный сине-чёрный) | заголовки, цифры |
| Текст вторичный | `#64748B` | подписи, описания |
| **Акцент (primary)** | `#16A34A` (зелёный Logip) | кнопки, активный пункт меню, прогресс |
| Инфо/графики | `#3B82F6` синий · `#F59E0B` янтарный · `#8B5CF6` фиолетовый | серии графиков, статусы |
| Статусы | зелёный = завершено · синий = в работе · серый = в ожидании · красный = просрочено | весь кабинет |
| Шрифт | **Inter** (полная кириллица), цифры — tabular-nums | весь UI |
| Гант: выполнено | заливка акцентным зелёным | полоса проекта |
| Гант: осталось | штриховка серым по белому | полоса проекта |

Витрина компонентов будет на служебной странице `/styleguide` — посмотришь вживую и скорректируем.
