## 1. ASCII-схема страницы «Пользователи» (t08, третья из трёх)

```
┌─ Пользователи ─────────────────────────────────────┬───────────────┐
│                                                    │ [+ Пригласить]│
│ ┌────────────────────────────────────────────────────────────────┐ │
│ │ Email            Имя      Роль       Статус       Действия     │ │
│ ├────────────────────────────────────────────────────────────────┤ │
│ │ gram2.0ai@…      Олег     Owner      ● активен    —            │ │
│ │ ivan@…           Иван     Employee   ● активен    [роль ▾] [🗑]│ │
│ │ new@…            —        Client     ○ приглашён  [⧉ ссылка]   │ │
│ │                                                   [↻ инвайт][🗑]│ │
│ └────────────────────────────────────────────────────────────────┘ │
└────────────────────────────────────────────────────────────────────┘

Диалог «Пригласить»:  Email* · Имя · Роль (select: Admin — только у Owner;
Employee; Client)  →  [Создать приглашение]
→ результат: ссылка установки пароля + кнопка «Копировать»
  (почтовой рассылки в первой очереди НЕТ — ссылку отправляет сам Owner/Admin)
```

Правила (из матрицы прав спеки 2.1):
- страница видна и доступна только Owner/Admin (guard уже стоит);
- Owner в таблице неприкосновенен (нет действий); себя удалить нельзя;
- Admin не может назначать роль Admin (только Owner); Admin не трогает Owner;
- «приглашён» = пользователь создан без пароля; [↻ инвайт] выпускает новую ссылку
  (старая гаснет), [⧉ ссылка] — скопировать действующую нельзя (хранится только хэш) —
  показывается одноразово при создании/перевыпуске.

## 2. Инвайт-флоу (t10) — решения

- Токен: 32 случайных байта (hex), в БД — только SHA-256-хэш; срок 7 дней; одноразовый.
- При создании инвайта заводится строка users (status=invited, без пароля); set-password
  по валидному токену ставит hash и переводит в active, гасит инвайт, сразу логинит.
- Повторный инвайт: старые неиспользованные токены email'а удаляются.
- Инвайт на email активного пользователя — отказ «пользователь уже активен».
- Пароль: минимум 8 символов, два поля должны совпадать.

## 3. Страница «Пользователи» (t09) — server actions с RBAC

`inviteUser` / `reinvite` / `changeRole` / `deleteUser` — каждое действие проверяет на
сервере: manageUsers(actor), canModifyUser(actor, target), assignAdmin для роли admin,
запрет самоудаления. UI-ограничения — только поверх серверных проверок.
