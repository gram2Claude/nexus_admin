## Принятые ограничения (документировано, без правок)

| Находка | Решение |
|---|---|
| schtasks без /RU: задача с интерактивным токеном не переживает reboot до логина | конвенция этого сервера (WGP-синк и timechecker живут так же; RDP-пользователь постоянно залогинен); симптом виден по жёлтой метке свежести |
| Метка свежести обновляется при загрузке страницы, не при soft-навигации | приемлемо для первой очереди; клиентский поллинг — вместе с обзором эпохи 4 |
| `normStatus` маппит неизвестные статусы в todo (cancelled появится — уточнить roll-up) | в текущих канонах только todo/in_progress/done |
| Путь node зашит в cmd-wrapper; лог без ротации | перерегистрация при смене node; лог мал (1 строка/прогон) |

## Чисто (проверено обоими)

Идемпотентность upsert и «реанимация» archived при возврате в канон; перенос задач между
спринтами; транзакционность per-project; manual-описания не упоминаются в синке вообще;
SQL параметризован; RBAC в actions; pct() с cap 100 и NULL-надёжностью; getFreshness не
роняет кабинет; cmd-wrapper решает quoting schtasks.
