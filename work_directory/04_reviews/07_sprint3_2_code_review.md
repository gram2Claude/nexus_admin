## Принято как ограничение (документировано)

- **Контракт ts-строк timechecker**: `::timestamptz` корректен, пока строки несут смещение
  (сейчас несут); строка без офсета интерпретировалась бы в TZ сессии. Попутно найден баг в
  самом timechecker (string-min по строкам с разными офсетами, claude.py:99) — кандидат в
  бэклог timechecker, на текущие данные не влияет.
- `levelFactRollup` возвращает Map только по спринтам с фактом — потребитель дефолтит нули
  (задокументировано в fact.ts).
